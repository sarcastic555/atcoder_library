Detailed Description

template<typename T, typename F>
class LazySegmentTree< T, F >

遅延セグメントツリー

条件

  • 集合T、演算op、単位元eがモノイドである
  • 集合Tから集合Tへの写像の集合Fが以下の条件を満たす
    • Fは恒等写像を含む
    • Fが閉じている
    • Fの任意の要素fに対して、f(x op y) = f(x) op f(y) が成立する

参考 https://tsutaj.hatenablog.com/entry/2017/03/30/224339 クエリ処理がAtCoder公式のライブラリより数倍遅く、以下の問題がTLEになってしまう
https://atcoder.jp/contests/practice2/tasks/practice2_k
公式のライブラリのコンセプトを使って将来的にリファクタをした方が良いかも
https://github.com/atcoder/ac-library/blob/master/atcoder/lazysegtree.hpp

コンストラクタ（初期ノードはT型としてユーザー側で初期化したものを渡す想定）

初期ノードと単位元が異なるケースを想定して作られたコンストラクタ
区間の合計を取得したい場合などは単位元と初期ノードが異なるのでこのコンストラクタを使う
例: 区間に対して加算クエリが来る状況下で区間の合計を取得する場合
合計を取得する場合は（各ノードにおける合計の反映のさせ方が子ノード数に依存するので）子ノードのサイズをデータとして持つ構造体をノードデータとする

struct NodeType {
int value;
int size;
};
NodeType f(int a, NodeType b) { return NodeType{b.value+a*b.size, b.size};}
int id = 0;
NodeType e{0, 0};
int IntSum(const int & a, const int & b) {return a+b;}
NodeType MyIntSum(const NodeType & a, const NodeType & b) { return NodeType{a.value+b.value, a.size+b.size}; };
std::vector<NodeType > init_vec(10, NodeType{0, 1});
LazySegmentTree<NodeType, int> tree(init_vec, MyIntSum, e, f, IntSum, id);

コンストラクタ（初期ノードは単位元を使う想定）

最下層ノードの初期値は単位元となる
使い方の例
区間に対して値の一括更新クエリが来る状況下で区間に対する最大値を求めたい

int e = 0;
int f(int a, int b) {return a;}
int id = 0;
int IntMax(int a, int b) {return std::max(a, b);}
LazySegmentTree<int, int> tree(10, IntMax, e, f, IntMax, id);
