Detailed Description

template<typename T, typename F>
class LazySegmentTree< T, F >

遅延セグメントツリー

クエリ処理がAtCoder公式のライブラリより数倍遅く、以下の問題がTLEになってしまう
https://atcoder.jp/contests/practice2/tasks/practice2_k
公式のライブラリのコンセプトを使って将来的にリファクタをした方が良いかも
https://github.com/atcoder/ac-library/blob/master/atcoder/lazysegtree.hpp

コンストラクタ（初期ノードは単位元を使う想定） / 

  最下層ノードの初期値は単位元となる  
  使い方の例  
  区間に対して値の一括更新クエリが来る状況下で区間に対する最大値を求めたい  
  int e = 0;  
  int f(int a, int b) {return a;}  
  int id = 0;  
  int IntMax(int a, int b) {return std::max(a, b);}  
  LazySegmentTree<int, int> tree(10, IntMax, e, f, IntMax, id);  

  @param[in] n ノードのサイズ
  @param[in] op ノード間の演算関数
  @param[in] e ノードの単位元
  @param[in] f ノードに適用される写像関数 (==の演算子が定義されている必要あり)
  @param[in] fg 写像間の合成関数 (右の写像の後にに左の写像を作用させる)
  @param[in] id 恒等写像
